Boundless Limousine – Full Stack Booking Form Challenge
This repository contains my submission for the Boundless Limousine Full Stack Web Developer mandatory skills test. The project is a responsive, high-performance booking form integrated with the Google Maps API and a mock backend for user data persistence.

🚀 Features
Pixel-Perfect Layout: Recreated the specific design from the provided reference.

Google Maps Integration: Real-time calculation of distance and travel time using the Google Maps Distance Matrix API.

Smart User Recognition: * The form checks the entered phone number against a mock database.

New Users: Prompts for First Name, Last Name, and Email.

Returning Users: Displays a personalized greeting (e.g., "Welcome back, Erick!").

Responsive Design: Fully optimized for mobile, tablet, and desktop views.

Form Validation: Robust client-side validation for emails, phone numbers, and required fields.

Mock API Submission: Data is dispatched to a simulated endpoint to demonstrate full-stack flow.

🛠️ Installation & Setup
Clone the repo:

Bash
git clone https://github.com/ericksonvilleta/boundless-limo-test.git
Install dependencies:

Bash
npm install
Environment Variables:
Create a .env file and add your Google Maps API Key:

Code snippet
REACT_APP_GOOGLE_MAPS_API_KEY=your_key_here
Run the project:

Bash
npm start
📝 Implementation Details
Distance Logic: Used the Distance Matrix service to ensure precise travel calculations between the pickup and drop-off points.

State Management: Managed user recognition logic to ensure a seamless "returning customer" experience without unnecessary page reloads.
